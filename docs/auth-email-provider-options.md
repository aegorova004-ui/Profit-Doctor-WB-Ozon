# Email/OTP provider options

Обновлено: 21 июля 2026 года.

Цель документа — выбрать провайдера для отправки login code в Profit Doctor WB/Ozon. Это не финальное решение и не интеграция: перед подключением нужно проверить регистрацию, оплату из России, договор/закрывающие документы, DNS-настройки домена и лимиты.

## Критерии выбора

1. Работает для российских пользователей и российского бизнеса.
2. Есть транзакционная отправка через API или SMTP.
3. Поддерживает доменную аутентификацию: SPF, DKIM, DMARC.
4. Можно отправлять короткие login-code письма без маркетинговой подписки.
5. Есть понятная статистика доставки и ошибок.
6. Можно хранить ключи только в env/secrets и не логировать plaintext-коды.

## Рекомендация для следующего шага

Начать с UnisenderGo как первого кандидата: он явно позиционируется как сервис транзакционных писем, подключается по SMTP/API и декларирует работу в России/Беларуси с серверами в РФ. Если регистрация, оплата или API-условия не подходят, сравнить с Sendsay Транспорт.

## Как подключать безопасно

1. Добавить env-переменные только в `.env.example`, без реальных ключей:
   - `AUTH_EMAIL_PROVIDER`;
   - `AUTH_EMAIL_FROM`;
   - `AUTH_EMAIL_API_KEY` или SMTP-настройки.
2. Реализовать provider adapter за интерфейсом `LoginCodeDelivery`.
3. Не возвращать login code клиенту и не писать его в логи.
4. До отправки проверять rate-limit.
5. Покрыть adapter unit-тестами через mock transport.
6. Перед production отправить тест на реальные почтовые ящики Mail.ru, Yandex, Gmail и проверить попадание во «Входящие».

# Sprint 2: Подтверждённые WB/Ozon-форматы

План создан 21 июля 2026 года. Статус: **запланирован; старт после приёмки Sprint 1 и решения по источникам эталонных форматов**.

## Цель

Перевести импорт из preview-режима к одному подтверждённому формату Wildberries и одному подтверждённому формату Ozon, не принимая неизвестные отчёты «на удачу» и не сохраняя чувствительные данные без auth/owner checks.

## Входные условия

- Sprint 1 принят пользователем: демо WB XLSX, WB CSV, Ozon CSV, себестоимость, диагноз, экспорт и мобильная таблица проверены.
- Есть один из безопасных источников формата для каждого маркетплейса:
  - полностью анонимизированный реальный отчёт;
  - официально подтверждённый шаблон;
  - synthetic production-like fixture, явно помеченный как preview, если реальный формат недоступен.
- Решение D-008 остаётся в силе: исходные отчёты и история расчётов не сохраняются на сервере.

## Срез 1 — карта форматов и диагностика

- [x] Составить таблицу поддерживаемых форматов: WB XLSX preview, WB CSV preview, Ozon CSV preview, будущие production-версии.
- [x] Для каждого формата зафиксировать обязательные колонки, необязательные расходы, правила знаков, ограничения и контрольные ожидания.
- [x] Разделить публичные ошибки на категории: неподдерживаемый формат, не хватает колонок, неверная валюта, неверный знак, неподдерживаемая операция, service-only отчёт.
- [x] Добавить UX-текст «что сделать дальше» для каждой категории ошибки.

Результат среза: [`docs/import-diagnostics.md`](import-diagnostics.md).

## Срез 2 — устойчивость парсеров

- [x] Добавить parser contract tests для всех поддерживаемых preview-версий.
- [ ] Проверить дубли колонок, пустые строки, BOM, `;`/`,`/tab-разделители и большие файлы.

- [ ] Уточнить политику предупреждений: что можно пропускать, а что обязано останавливать импорт.

## Срез 3 — UX импорта неизвестного отчёта

- [ ] Показать пользователю распознанный маркетплейс и версию адаптера до результата.
- [ ] Для неизвестного CSV/XLSX показать понятную карточку: какие колонки найдены, каких не хватает, какой шаблон скачать.
- [ ] Добавить ссылку на демо-шаблоны прямо из ошибки неподдерживаемого формата.
- [ ] Не выводить сырые строки отчёта, ФИО, телефоны, email, названия кабинетов или другие чувствительные поля.

## Срез 4 — подготовка к production-адаптерам

- [ ] Создать пустые версии контрактов `wb:<confirmed-format>:YYYY-MM` и `ozon:<confirmed-format>:YYYY-MM` только после безопасного подтверждения схемы.
- [ ] Для каждого production-контракта добавить fixture, ожидаемые агрегаты, warnings/errors и ручной контрольный расчёт.
- [ ] Обновить `docs/finance-formulas.md`, если production-формат приносит рекламу, хранение, налоги или себестоимость из отчёта.
- [ ] Не менять статус preview на production без подтверждённого источника формата.

## Критерий готовности Sprint 2

Пользователь может загрузить по одному подтверждённому формату WB и Ozon; приложение корректно определяет версию, считает прибыль по SKU, объясняет отсутствующие данные, безопасно отказывает неизвестным отчётам и не сохраняет исходные файлы на сервере.

## Что проверить пользователю

- Достаточно ли понятны ошибки неизвестного формата.
- Нужна ли ручная загрузка себестоимости отдельным CSV или пока хватает ввода на странице.
- Приемлем ли preview-режим для Ozon до появления реального анонимизированного отчёта.

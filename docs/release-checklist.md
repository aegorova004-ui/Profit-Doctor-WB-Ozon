# Checklist релиза

Заполняется для каждого production-релиза. Неприменимый пункт помечается `N/A` с причиной.

## Объём и решение

- [ ] Описаны пользовательская ценность, границы релиза и критерии готовности.
- [ ] Все изменения связаны с Issue/PR; `CHANGELOG.md`, `PROJECT_STATE.md` и нужные документы обновлены.
- [ ] Для миграций, платежей и рискованных изменений есть rollback plan.
- [ ] Нет незакрытых критичных замечаний review.

## Проверки

- [ ] Для бесплатного static-slice на GitHub Pages проверены `/`, `/upload`, демо-отчёт, ручная загрузка и экспорт.
- [ ] `npm ci` проходит на чистом окружении.
- [ ] `npm run db:generate` и `npm run db:validate` проходят.
- [ ] `npm run format:check`, `npm run typecheck` и `npm run lint` проходят.
- [ ] `npm test` и `npm run build` проходят.
- [ ] Пройден e2e smoke: вход → загрузка → прибыльность → проблемные SKU → экспорт.
- [ ] Ошибки и пустые состояния проверены вручную.

## Финансы и импорт

- [ ] Изменённые формулы описаны и покрыты unit-тестами с ручными контрольными примерами.
- [ ] Маржа/ROI не вычисляются при нулевом знаменателе; неполные результаты помечены как оценка.
- [ ] Версии адаптеров и fixtures соответствуют поддерживаемым форматам.
- [ ] Публичные demo fixtures соответствуют manifest-тесту; в `public/demo/` нет устаревших дубликатов.
- [ ] Возвраты, сторно, дубли, знаки удержаний и округление проверены.
- [ ] Пользователь видит происхождение сумм и отсутствующие компоненты.
- [ ] CSV/XLSX-экспорт не исполняет пользовательские SKU, артикулы и названия товаров как spreadsheet formulas.

## Безопасность и данные

- [ ] В репозитории, логах и fixtures нет секретов, персональных данных и реальных отчётов.
- [ ] Загрузка проверяет владельца, размер, MIME-тип, расширение и структуру файла.
- [ ] Данные одного пользователя нельзя получить по идентификатору другого.
- [ ] Срок хранения исходных файлов и удаление проверены.
- [ ] Платёжный статус подтверждается на сервере; webhook защищён от повтора и подделки.

## Миграция и эксплуатация

- [ ] Production-переменные заданы в secrets; тестовые ключи не используются.
- [ ] Миграция проверена на копии схемы; резервная копия создана и восстановление понятно.
- [ ] Хостинг, домен, TLS и эквайринг доступны из России.
- [ ] После выкладки проверены health/status, основной сценарий и отсутствие всплеска ошибок.
- [ ] Назначены ответственный за наблюдение и окно rollback.

## После релиза

- [ ] Версия/тег и changelog опубликованы.
- [ ] Проверены первые реальные импорты без просмотра чувствительного содержимого.
- [ ] Обратная связь и найденные дефекты заведены в Issues.
- [ ] `PROJECT_STATE.md` обновлён фактическим результатом.

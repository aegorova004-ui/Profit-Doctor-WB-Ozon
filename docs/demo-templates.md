# Демо-шаблоны отчётов

Обновлено: 21 июля 2026 года.

Все файлы ниже полностью синтетические. Они придуманы для проверки интерфейса, парсеров, ошибок и экспорта Profit Doctor. В них нет данных продавцов, покупателей, кабинетов, ИП, ООО, телефонов, email или настоящих операций.

Файлы лежат в `public/demo/` и доступны локально из запущенного сайта по адресам `/demo/...`.

Единственный источник правды для путей — `src/domain/reports/demo-fixtures.ts`. Пользовательский UI показывает одно базовое демо, а остальные шаблоны используются для ручной QA-проверки и manifest-теста, чтобы пути не расходились между собой.

## Рабочие шаблоны

| Файл                               | URL на локальном сайте                   | Что проверяет                                                                                                                                 |
| ---------------------------------- | ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `wb-financial-report-preview.xlsx` | `/demo/wb-financial-report-preview.xlsx` | Рабочий WB XLSX preview: продажи, возврат, логистика, хранение, ручная себестоимость, диагноз, экспорт.                                       |
| `wb-finance-api-preview.csv`       | `/demo/wb-finance-api-preview.csv`       | Рабочий WB finance CSV preview: `supplier_oper_name`, продажи, возврат, дополнительные удержания, сервисная строка без SKU с предупреждением. |
| `wb-finance-large-preview.csv`     | `/demo/wb-finance-large-preview.csv`     | Большой WB finance CSV preview: 36 товарных SKU для проверки прокрутки, мобильных карточек, повторного пересчёта и экспорта.                  |
| `ozon-finance-preview.csv`         | `/demo/ozon-finance-preview.csv`         | Рабочий Ozon finance CSV preview: продажи, возврат, комиссия, логистика, обработка отправления и прочие услуги.                               |

## Шаблон для проверки ошибки

| Файл                                  | URL на локальном сайте                      | Что проверяет                                                                                                               |
| ------------------------------------- | ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `wb-product-catalog-not-finance.xlsx` | `/demo/wb-product-catalog-not-finance.xlsx` | Негативный сценарий: файл похож на товарный каталог WB, а не на финансовый отчёт. UI должен объяснить это понятной ошибкой. |
| `unsupported-finance-format.csv`      | `/demo/unsupported-finance-format.csv`      | Негативный сценарий: CSV содержит несколько похожих полей, но не является поддерживаемым финансовым отчётом WB/Ozon.        |

## Что считается успешной проверкой

1. Кнопка «Посмотреть демо на примере» открывает `wb-financial-report-preview.xlsx` и показывает 4 операции, 3 SKU.
2. `wb-finance-api-preview.csv`, если скачать и выбрать вручную, показывает 3 операции, 2 SKU, а также предупреждение про сервисную строку без SKU.
3. `wb-finance-large-preview.csv`, если скачать и выбрать вручную, показывает 36 операций и 36 SKU; страница должна свободно прокручиваться, а мобильная версия должна показывать карточки без горизонтального чтения.
4. `wb-product-catalog-not-finance.xlsx`, если выбрать его вручную, не считается отчётом и показывает ошибку: «Похоже, выбран товарный каталог Wildberries».
5. `unsupported-finance-format.csv`, если выбрать его вручную, показывает карточку диагностики: найденные колонки, недостающие поля и ссылки на демо-шаблоны.
6. `ozon-finance-preview.csv`, если скачать и выбрать вручную, показывает 4 операции, 3 SKU.

## Автоматическая проверка

```bash
npm test -- src/domain/reports/demo-fixtures.test.ts
```

Тест проверяет, что в `public/demo/` нет лишних файлов, рабочие шаблоны не пустые и разбираются нужными preview-адаптерами, а негативные шаблоны остаются понятными ошибками.

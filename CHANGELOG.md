# Журнал изменений

Формат основан на Keep a Changelog. Пока нет первого релиза, изменения собираются в разделе «Не выпущено».

## Не выпущено

### Добавлено

- Базовый проект Next.js 16 с TypeScript и зависимостями Prisma/PostgreSQL.
- Landing, auth skeleton и локальная проверка CSV/XLSX до 10 МБ без серверной отправки.
- Prisma schema и начальная миграция для пользователей, отчётов, строк, снимков прибыльности и тарифов.
- Финансовое ядро в integer-копейках, контракт парсера, определение WB/Ozon и синтетические fixtures.
- 21 unit-тест финансовых формул, определения маркетплейса и проверки файла.
- Проектная документация, план `Sprint 0: Foundation` и журнал архитектурных решений.
- CI для установки зависимостей, Prisma, форматирования, типизации, lint, тестов и сборки.
- Автоматические GitHub Issues для недельного планирования и вечерних отчётов.
- Шаблоны pull request и задачи Codex.
- Оригинальный маскот Profit Doctor с подсказками по ходу landing.
- Растровые PNG-ассеты демо-диагностики и пиксельного маскота, включая отдельный кадр для дружелюбного взмаха рукой.
- Синтетический XLSX-fixture финансового отчёта Wildberries с полями из официальной документации API.
- Preview-адаптер `wb:api-financial-report:preview-2026-07` для локального чтения XLSX, агрегации продаж и возвратов по SKU и сверки с `for_pay`.
- Точная десятичная обработка сумм через строки и `BigInt`, без промежуточных JS float.
- Локальная сводка импорта и таблица SKU с честной оценкой до себестоимости и рекламы.
- Integration-тест загрузки бинарного XLSX через React-компонент без сетевых запросов.
- Ручной ввод себестоимости единицы по SKU с точным пересчётом через integer-копейки и `BigInt`.
- Расшифровка источников сумм и статусы «Убыток», «Пока в плюсе» и «Нужна себестоимость» в таблице анализа.
- Диагноз по SKU с разрывом до безубыточности, предельной себестоимостью до рекламы, приоритетами и крупнейшей известной статьёй расходов.
- Документ `docs/auth-owner-checks.md`, Prisma-модели `LoginCode`/`AuthSession`, auth-token helpers, login-code flow, session resolver, session-cookie policy и server access-control guards для будущей истории отчётов.

### Изменено

- Формула прибыли собрана в читаемое уравнение «выручка − расходы = прибыль по SKU» с расшифровкой расходов.
- Увеличены кегль и контраст текста в списке проблем, карточках аудитории и тарифных возможностях.
- Тарифы теперь явно наследуют возможности предыдущего уровня; у каждого пункта есть доступная информационная подсказка.
- Демо-диагностика на первом экране заменена цельной прямой PNG-карточкой без составной вёрстки и перекошенных слоёв.
- Маскот поднимает руку на время наведения на десктопе и один раз на 700 мс после тапа на планшете или смартфоне.
- Страница `/upload` теперь запускает первый WB XLSX-анализатор в браузере вместо неактивной кнопки Sprint 1.
- Сводка импорта разделяет удержания WB, введённую себестоимость и результат до рекламы; положительная неполная оценка больше не выглядит как подтверждённая прибыль.

### Исправлено

- Текст во всех трёх карточках аудитории начинается на одном уровне.
- Карточка Start снова получает выделенный фон и сохраняет общую геометрию тарифной сетки.
- Подсказка тарифа Start поднимается поверх соседних карточек; на десктопе открывается наведением, на сенсорных устройствах — нажатием.
- Кнопка пересчёта себестоимости теперь показывает ошибку пустой формы или подтверждение успешного обновления результата.
- Страница импорта сохраняет вертикальную прокрутку после пересчёта; автоматическая плавная прокрутка удалена.
- Таблица результата полностью помещается на широком экране, а в узком окне и на мобильном превращается в читаемые вертикальные карточки.

### Безопасность

- Зафиксированы правила работы с отчётами продавцов, секретами и трассировкой финансовых расчётов.
- Принято решение D-008: серверная история отчётов запрещена до auth, owner checks и тестов на чужие данные.
- `npm audit` проходит без известных уязвимостей; транзитивный PostCSS закреплён на исправленной версии.
